Staff Member Portal

A mobile-first staff portal for The 1898 Niseko boutique hotel, enabling 6 team members to manage daily tasks, update status, communicate with supervisors, and deliver exceptional guest experiences.

Executive Summary

Route Strategy

Mobile-First Design

Optimized for one-handed use, quick status updates, and photo capture on mobile devices

Core Workflow

Dashboard → Task Details → Complete with Photo → Update Status

Feature Requirements

US-01

Staff Member Dashboard

As a hotel staff member, I want to see my personal dashboard with avatar, current status, and today's tasks so I can quickly understand my daily responsibilities.

Acceptance: Dashboard shows staff avatar, name, role, current status badge, task count, and quick action buttons
US-02

Task Detail & Completion

As a housekeeper, I want to view task details including room number and requirements, then complete tasks with photo proof so my supervisor can verify quality.

Acceptance: Task detail page shows full information, includes mobile photo capture, allows status change, and validates completion
US-03

Status Management

As a concierge, I want to update my status between available, busy, on break, and off duty so the team knows my availability for guest requests.

Acceptance: Status selector with 4 states, visual indicators, timestamp tracking, and real-time updates visible to supervisors
US-04

Issue Reporting

As a maintenance staff member, I want to report issues with photos and descriptions so urgent problems are communicated quickly to supervisors.

Acceptance: Issue form with category, priority, photo upload, description field, and confirmation of supervisor notification

Data Models

StaffMember Type

interface StaffMember {
  id: string;                    // Unique identifier
  name: string;                  // Full name
  role: string;                  // Job title (Housekeeper, Concierge, etc.)
  department: string;            // Housekeeping, Front Desk, Kitchen, etc.
  avatar: string;                // Avatar image path
  status: StaffStatus;            // Current availability status
  email: string;                 // Contact email
  phone: string;                 // Contact phone
  tasks: string[];               // Array of task IDs assigned today
  supervisor: {                   // Supervisor info for escalation
    name: string;
    avatar: string;
    agentId: string;
  };
  shiftStart?: string;           // Today's shift start time
  shiftEnd?: string;             // Today's shift end time
}

type StaffStatus = 'available' | 'busy' | 'on-break' | 'off-duty';

Extended Task Type (for Staff View)

interface StaffTask extends TaskData {
  completionPhotos?: string[];   // Base64 or URLs of completion photos
  estimatedDuration?: number;    // Minutes (for planning)
  roomNumber?: string;           // Specific room if applicable
  guestNotes?: string;           // Special guest preferences
}

• Large touch targets (min 44x44px)
• Bottom-fixed action buttons
• Collapsible task details
• Quick-access status toggle

• Persistent sidebar navigation
• Expanded task cards
• Inline photo preview
• Hover states and tooltips

Photo Upload Workflow

// Mobile Camera Integration (Mocked for MVP)
<input
  type="file"
  accept="image/*"
  capture="environment"  // Triggers rear camera on mobile
  onChange={handlePhotoCapture}
/>

// Base64 encoding for display (no upload API yet)
const handlePhotoCapture = (e: ChangeEvent) => {
  const file = e.target.files?.[0];
  if (file) {
    const reader = new FileReader();
    reader.onloadend = () => {
      const base64 = reader.result as string;
      setCompletionPhotos([...completionPhotos, base64]);
    };
    reader.readAsDataURL(file);
  }
};
